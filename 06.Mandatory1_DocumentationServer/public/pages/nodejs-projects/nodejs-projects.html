Running Node.js Projects – node CLI, nodemon, executing files
How do you set up a website & run it ? For us developers to run node we use the node REPL, with the command "node fileName.js" at the correct path, or through ../folder/fileName.js. You can also choose to download a tool "npm install nodemon", as it is global you can use it wherever you want, it is the same syntax, "nodemon fileName.js".

npm and Project Setup – package.json, dependencies, folder structure
Nodejs uses 'npm' as a build tool that you can use to install dependencies in a package.json file for your websites in a package/folder. To setup use the command "npm init" then "npm install express", or you could set up an package.json yourself & then tell it to install its dependencies through "npm install" when setting up local websites. 

{ "dependencies": { "express" : "5.2.1" } }

These dependencies are downloaded into node_modules together with other automatically configured data to get nodejs to run on this operating system, that package acts as a library for the website. Each of the other fileName.js is also known as a module in nodejs. There are other build tools other than npm, like pip for python, maven & gradle for java. There lies a difference between those different build tools. But for the most part they are acting for the most part a package manager like npm, pip. But some also enforce a structure you have to follow like maven. But most have a file which contians dependencies like package.json. You choose based on the project size, familiarty of use to the team, and the structure & ecosystem. I chose nodejs with npm as it is more flexible and enforces less upon me than for example maven and its lifecycle structure.
