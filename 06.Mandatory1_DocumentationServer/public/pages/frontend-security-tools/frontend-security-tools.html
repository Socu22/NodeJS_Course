XSS Prevention – Sanitizing input, innerHTML vs textContent
For prevention of Cross-Site Scripting (XSS) when displaying user input on a webpage. This make the page secure when using .innerHTML, if you want to prevent using this function then use .innertext or .textContent & avoid using the vulnable .innerhtml. 

                function sanitizeXSS(string) {
                    return string
                        .replaceAll('&', '&')
                        .replaceAll('<', '<')
                        .replaceAll('>', '>')
                        .replaceAll('"', '"')
                        .replaceAll("'", ''')
                        .replaceAll('`', '`')
                        .replaceAll('/', '/');
                }
                const userComment = '<script>alert("Hacked!")</script>';
                const safeComment = sanitizeXSS(userComment);
                document.getElementById("comment").innerHTML = safeComment;
