Functions & Callbacks – First-class functions, hoisting, arrow functions Functions are first-class citizens which means they come first compared to everything else, all functions are hoisted to be instantiated in the beggining automatically no matter where/when in the code it is instantiated. So other code can use it in whichever sequence you want. One of the important ones is the arrow function used in callback functions. Callback function is a function thats referenced by another as an argument function genericActionPerformer(number, action) or like this function genericActionPerformer(number, () => {functionLogic})

Functional Array Methods – map, forEach, filter, reduce, side-effect handling Functional coding, when looping through an array you usually use: map .forEach .filter .reduce .sort .find .findIndex .indexOf, ect. When reading use .forEach unless you need the data afterwerwards then use .map as it has fewer side effects and actually returns a 1:1 new array, instead of having to create a new const for the looped data.
This requires a ternary statement "condition ? if true : if false" & a spread operator "...objects" to avoid side effects if the elements in the array are objects. Objects in a array are only refrenced to be there from memory which means a if statement won't be enough. The side effects here is that even if you make a new const with newObjects.map((object) => ({if(condition) then return obejct} ) then it will change the orginalObjects as well. you need to manually send each object to the newObjects with newObjects.map((object) => {keyNumber: condition ? ifTrueThen : ifFalseThen, ...object }) )
