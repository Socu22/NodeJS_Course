SSR (Server-Side Rendering) – Pre-rendered HTML, client-server model


This means that based on the client-server model, the client receives a complete html, and from the server's side it generates the complete html before sending it to the client. There is a difference between that compared to receiving an unrendered empty html which the client needs to render. This will limit who will be able to use the site as the client will need to use its own resources. Keep in mind that the previous site used client-side Rendering. Now we show an example of Server-Side Rendering in an ES module express site. It also considers components which need to be put together in every html or most of them. It follows the burger principle, with the main element being the middle and the two other parts being the header & footer of the html. MainPackage --> public --> components --> component --> html

templateEngine.js – Combine header, page, footer with placeholders


Using the previous html's we make a function which combines the two buns with the meat. At the same time also uses $$DOCUMENT_TITLE$$ & $$CSS_LINKS$$ as placeholders for title & stylesheet.
It is a part of the MainPackage --> public --> util --> script. 

import fs from 'fs';
export function constructPage(page, options = {}) { // options can use the key-value pairs inside its object.
    const header = readPage('./public/components/header/header.html');
    const footer = readPage('./public/components/footer/footer.html');

    return header
            .replace('$$DOCUMENT_TITLE$$', options.documentTitle || 'unnamed site')
            .replace('$$CSS_LINKS$$', options.cssLinks || '')
         + page 
         + footer;
}
export function readPage(path) {// Synchronous file-reading to buffer to string. as it is html being read, it returns html. 
    return fs.readFileSync(path).toString(); 
}

pagesRouter.js – Server-side rendered routes

Instead of having clutter inside app.js in the form of routes then we make this which is a part of the
MainPackage --> routers --> pagesRouter.js
We inport the pagePage which is the page which is server rendered ready for the client. The Router is equal to the current express backend. 

import { Router } from "express";
import { pagePage } from "../util/pagesUtil.js";
const router = Router();
router.get("/", (req, res) => {
  res.send(nodejsBasicsPage); // the fully rendered html during server side Rendering.
});
export default router;
